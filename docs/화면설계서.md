## 1. 시스템 개요

### 1.1 서비스 개요

GameCoach는 게임 코치(강사)와 수강생을 연결하는 1:1 코칭 마켓플레이스입니다. MetaMask 지갑 인증을 통해 가입하고, Ethereum 스마트 컨트랙트 에스크로로 결제 신뢰성을 보장합니다.

### 1.2 사용자 역할

| 역할 | 설명 | 주요 화면 |
|------|------|-----------|
| 비회원 (Guest) | 미로그인 사용자 | 랜딩, 코치 목록/상세, 로그인, 회원가입 |
| 수강생 (Student) | 코치에게 수업 신청 | 코치 목록/상세, 수강생 대시보드, 채팅, 리뷰 |
| 코치 (Coach) | 수업 요청 수락·관리 | 코치 대시보드, 채팅 |

### 1.3 인증 방식

MetaMask EIP-191 서명 인증:  
1. `eth_requestAccounts` → 지갑 주소 획득  
2. `/api/auth/nonce` → 일회성 nonce 발급 (KV 저장, 5분 만료)  
3. `personal_sign(message, address)` → 서명 생성  
4. `/api/auth/wallet` → 서명 검증 (ethers.verifyMessage) → 세션 생성  
5. `gc_session` HttpOnly 쿠키에 세션 토큰 저장

### 1.4 결제 흐름

- **예약금(30%)**: 수업 신청 시 MetaMask 트랜잭션으로 에스크로 컨트랙트에 선납
- **잔금(70%)**: 코치 수락 후 수업 완료 시 자동 정산
- **환불**: 코치 거절 시 예약금 즉시 반환

## 4. SCR-02 로그인 `/auth/login`

**파일**: `src/app/auth/login/page.tsx`

### 4.1 화면 구성

중앙 정렬 카드 (maxWidth: 400px)

| 요소 | 내용 |
|------|------|
| 로고 | "G" 아이콘 (ink 배경, accent 색상) |
| 제목 | "MetaMask로 로그인" |
| 서브텍스트 | "연결된 지갑 주소로 간편하게 로그인하세요" |
| MetaMask 아이콘 | 🦊 이모지 (80×80, ink 배경 원형) |
| 로그인 버튼 | "MetaMask로 로그인" (btn-primary btn-lg btn-block) |
| MetaMask 설치 안내 | notice 박스, metamask.io 링크 |
| 회원가입 링크 | "처음 방문이신가요? 회원가입" → `/auth/register` |

### 4.2 버튼 상태 (Step)

| 단계 | 버튼 텍스트 | 버튼 스타일 |
|------|-------------|-------------|
| idle | MetaMask로 로그인 | btn-primary |
| connecting | 지갑 연결 중... | sunken 배경, 스피너 |
| signing | MetaMask에서 서명해주세요... | sunken 배경, 스피너 |
| done | (자동 리다이렉트) | — |

### 4.3 에러 처리

| 에러 조건 | 표시 메시지 |
|-----------|-------------|
| MetaMask 미설치 | "MetaMask가 설치되어 있지 않습니다." |
| 계정 없음 | "지갑 계정을 찾을 수 없습니다. MetaMask에서 계정을 선택해주세요." |
| nonce 실패 | "nonce 발급 실패" |
| 미가입 지갑 (404) | "등록되지 않은 지갑입니다. 먼저 회원가입을 해주세요." + 회원가입 링크 |
| 기타 서버 오류 | API 응답의 error 메시지 |

에러는 danger-tint 배경의 notice 박스로 표시, warn 아이콘 포함

### 4.4 인증 플로우

1. 버튼 클릭 → `eth_requestAccounts` → `accounts[0]` 추출
2. `GET /api/auth/nonce?address={addr}` → nonce 획득
3. MetaMask `personal_sign` → signature 획득
4. `POST /api/auth/wallet` `{ address, signature }` → 세션 생성
5. 성공 시 → `/` 리다이렉트 (router.push)

### 4.5 상태 전환 다이어그램

```
[idle] → (클릭) → [connecting] → (accounts 획득) → [signing]
                                                         ↓ (서명 완료)
[done (→ 홈)] ← (API 성공) ← [signing 완료]
[idle (에러 표시)] ← (어느 단계에서든 실패)
```

## 13. API 명세

### 13.1 인증 API

#### `GET /api/auth/nonce`

**쿼리 파라미터**: `?address={wallet_address}`  
**응답**: `{ nonce: string }` 또는 `{ error: string }`  
**동작**: UUID nonce 생성 → KV에 `nonce:{address}` 키로 5분 TTL 저장

---

#### `POST /api/auth/wallet`

**요청 Body**:
```json
{
  "address": "0x...",
  "signature": "0x...",
  "role": "coach" | "student"  // 회원가입 시만
}
```

**동작**:
1. KV에서 nonce 조회 (만료 시 401 반환)
2. `ethers.verifyMessage(message, signature)` 로 서명 검증
3. 로그인: D1 users 테이블에서 address 조회, 없으면 404
4. 회원가입: D1에 신규 유저 INSERT (role, wallet, id)
5. KV에 세션 저장 (`session:{token}` 키)
6. `gc_session` HttpOnly 쿠키 설정

**응답**: `{ ok: true, role: string }`

---

#### `GET /api/auth/me`

**쿠키**: `gc_session` 필요  
**응답**:
```json
{
  "id": "uuid",
  "address": "0x...",
  "role": "coach" | "student",
  "nickname": "string"
}
```
미인증 시 401

---

#### `POST /api/auth/logout`

**동작**: KV에서 세션 삭제, `gc_session` 쿠키 만료  
**응답**: `{ ok: true }`

---

## 14. 데이터베이스 스키마

### D1 (Cloudflare SQLite)

```sql
-- 사용자
CREATE TABLE users (
  id         TEXT PRIMARY KEY,
  phone      TEXT UNIQUE,
  discord_id TEXT UNIQUE,
  role       TEXT NOT NULL CHECK(role IN ('coach','student','both')),
  wallet     TEXT UNIQUE,
  nickname   TEXT NOT NULL DEFAULT '',
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);

-- 코치 프로필
CREATE TABLE coaches (
  id            TEXT PRIMARY KEY REFERENCES users(id),
  game_category TEXT,
  tier          TEXT,
  tier_self     INTEGER DEFAULT 1,  -- 1: 자기신고, 0: 검증됨
  price_eth     TEXT DEFAULT '0.05',
  session_min   INTEGER DEFAULT 60,
  intro         TEXT DEFAULT '',
  avg_rating    REAL DEFAULT 0,
  review_count  INTEGER DEFAULT 0,
  updated_at    INTEGER NOT NULL DEFAULT (unixepoch())
);

-- 수업
CREATE TABLE lessons (
  id         TEXT PRIMARY KEY,
  coach_id   TEXT NOT NULL REFERENCES users(id),
  student_id TEXT NOT NULL REFERENCES users(id),
  slot       TEXT NOT NULL,          -- "2026-06-09 18:00"
  session_min INTEGER NOT NULL DEFAULT 60,
  price_eth  TEXT NOT NULL,
  state      TEXT NOT NULL DEFAULT 'PENDING'
             CHECK(state IN ('PENDING','ACCEPTED','ACTIVE','COMPLETED','REJECTED')),
  message    TEXT DEFAULT '',
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);

-- 메시지
CREATE TABLE messages (
  id         TEXT PRIMARY KEY,
  lesson_id  TEXT NOT NULL REFERENCES lessons(id),
  sender_id  TEXT NOT NULL REFERENCES users(id),
  body       TEXT NOT NULL,
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);

-- 리뷰
CREATE TABLE reviews (
  id         TEXT PRIMARY KEY,
  lesson_id  TEXT NOT NULL REFERENCES lessons(id),
  rating     INTEGER NOT NULL CHECK(rating BETWEEN 1 AND 5),
  tags       TEXT DEFAULT '[]',  -- JSON array
  body       TEXT DEFAULT '',
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);
```

### KV (Cloudflare KV)

| 키 패턴 | 값 | TTL |
|---------|----|-----|
| `nonce:{wallet_address}` | nonce 문자열 | 300초 (5분) |
| `session:{token}` | JSON `{ userId, wallet, role, nickname }` | 604800초 (7일) |

## 15. 인증 플로우

### 15.1 회원가입 전체 플로우

```
[/auth/register]
  ↓ 역할 선택 (student/coach)
  ↓ MetaMask 연결 (eth_requestAccounts → accounts[0])
  ↓ GET /api/auth/nonce?address={addr}
  ↓ personal_sign("GameCoach 인증 요청\n\nnonce: {nonce}", addr)
  ↓ POST /api/auth/wallet { address, signature, role }
      → D1 INSERT users
      → KV SET session:{token} (7일)
      → Set-Cookie: gc_session={token}
  ↓ router.push("/profile/setup")
  
[/profile/setup]
  ↓ 닉네임 (+코치: 자기소개, 카테고리, 티어, 가격, 시간) 입력
  ↓ PATCH /api/profile { nickname, ... }
      → D1 UPDATE users SET nickname
      → D1 INSERT/UPDATE coaches (코치인 경우)
  ↓ router.push("/")
```

### 15.2 로그인 전체 플로우

```
[/auth/login]
  ↓ MetaMask 연결
  ↓ GET /api/auth/nonce
  ↓ personal_sign
  ↓ POST /api/auth/wallet { address, signature }
      → D1에서 address 조회
      → 없으면 404 → "등록되지 않은 지갑" 에러
      → 있으면 세션 생성
  ↓ router.push("/")
```

### 15.3 로그아웃 플로우

```
[TopNav 드롭다운 → 로그아웃]
  ↓ POST /api/auth/logout
      → KV DELETE session:{token}
      → Set-Cookie: gc_session=; Max-Age=0
  ↓ 로컬 상태 초기화 (me=null, walletConnected=false)
  ↓ router.push("/") + router.refresh()
```
